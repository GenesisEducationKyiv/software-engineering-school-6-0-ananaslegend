# Docker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Containerize the app with a multi-stage Dockerfile and a docker-compose that starts postgres + mailpit + migrations + API with a single `docker compose up --build`.

**Architecture:** Multi-stage Dockerfile (builder=golang:1.26-alpine → runtime=alpine:3.21). docker-compose runs four services in order: postgres (with healthcheck) → migrate (exits 0) → api (depends_on service_completed_successfully). Mailpit runs in parallel with postgres.

**Tech Stack:** Docker, docker-compose v2, `migrate/migrate` image, `axllent/mailpit` image, `postgres:17-alpine`.

### Task 2: Dockerfile (multi-stage)

**Files:**
- Create: `Dockerfile`

- [ ] **Step 1: Create `Dockerfile`**

```dockerfile
FROM golang:1.26-alpine AS builder
WORKDIR /build
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o /api ./cmd/api

FROM alpine:3.21 AS runtime
RUN apk add --no-cache ca-certificates tzdata
COPY --from=builder /api /api
EXPOSE 8080
ENTRYPOINT ["/api"]
```

- [ ] **Step 2: Verify build succeeds**


### Task 3: docker-compose.yml

**Files:**
- Create: `docker-compose.yml`

- [ ] **Step 1: Create `docker-compose.yml`**

```yaml
services:
  postgres:
    image: postgres:17-alpine
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: postgres
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 10

  mailpit:
    image: axllent/mailpit:latest
    ports:
      - "8025:8025"

  migrate:
    image: migrate/migrate:latest
    volumes:
      - ./migrations:/migrations:ro
    command:
      - "-path=/migrations"
      - "-database=postgres://postgres:pass@postgres:5432/postgres?sslmode=disable"
      - "up"
    depends_on:
      postgres:
        condition: service_healthy
    restart: on-failure

  api:
    build: .
    ports:
      - "8080:8080"
    environment:
      DATABASE_URL: postgres://postgres:pass@postgres:5432/postgres?sslmode=disable
      APP_BASE_URL: http://localhost:8080
      SMTP_HOST: mailpit
      SMTP_PORT: "1025"
      SMTP_TLS_POLICY: none
      LOG_PRETTY: "true"
      LOG_LEVEL: info
    env_file:
      - path: .env
        required: false
    depends_on:
      migrate:
        condition: service_completed_successfully
      mailpit:
        condition: service_started

volumes:
  pgdata:
```

Note: `env_file.required: false` means compose works even if `.env` doesn't exist. When `.env` exists, its values override the inline `environment` block — useful for setting `CONFIRM_TOKEN_TTL`, `DB_MAX_CONNS`, etc.

```makefile

	docker compose up --build

	docker compose down

```

`docker-clean` removes the `pgdata` volume — use when you want a fresh database.
