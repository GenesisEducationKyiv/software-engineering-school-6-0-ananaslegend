# GitHub CI Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions CI workflow that runs `go vet`, `golangci-lint`, `go test`, and `go build` on every push and PR to `main`.

**Architecture:** Single linear job in one YAML file. Vendor mode means no module cache needed. All tests are unit tests — no services (postgres/redis) required in CI.

**Tech Stack:** GitHub Actions, `actions/checkout@v4`, `actions/setup-go@v5`, `golangci-lint-action@v6`, Go 1.26, `-mod=vendor`

---

### Task 1: Create the CI workflow file

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the directory and workflow file**

Create `.github/workflows/ci.yml` with the following content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    name: Lint, Test & Build
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: '1.26'
          cache: false

      - name: Vet
        run: go vet -mod=vendor ./...

      - name: Lint
        uses: golangci/golangci-lint-action@v6
        with:
          args: --mod=vendor

      - name: Test
        run: go test -mod=vendor ./...

      - name: Build
        run: go build -mod=vendor -o /dev/null ./cmd/api
```

- [ ] **Step 2: Verify the file structure**

```bash
cat .github/workflows/ci.yml
```

Expected: full YAML content as written above, no truncation.

- [ ] **Step 3: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions workflow (vet, lint, test, build)"
```

Expected: commit succeeds, 1 file changed.

---

### Task 2: Verify CI works on GitHub

- [ ] **Step 1: Push to remote**

```bash
git push origin main
```

- [ ] **Step 2: Check Actions tab**

Open the repository on GitHub → Actions tab. You should see the "CI" workflow running for the push commit.

Expected: all 4 steps (Vet, Lint, Test, Build) pass with green checkmarks.

- [ ] **Step 3: Confirm golangci-lint finds no issues**

If the Lint step fails with lint errors, fix the reported issues in the relevant files, then re-push. Common first-run findings: unused variables, missing error checks, `gofmt` drift.
